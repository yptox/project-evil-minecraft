V2 — CORRUPTION (active)   namespace AlgorithmicGallery.Corruption
==============================================================

DATA / MANIFEST
- CuratedPropManifest.cs   Loads curated-props.json (833 daily-life GLBs).
                           Random / weighted-by-tags / drifted-from-tags queries.
- StyleProfile.cs          Tracks player placement behavior — group/tag counts,
                           density grid, cadence. Read by AssistantSystem.

UI / INPUT
- HotbarController.cs      5-slot hotbar. 1-5 keys to select. Reroll on consume.
                           Assistant override probability (driven by AssistantSystem).
- HotbarUI.cs              Runtime UGUI canvas — 5 slot panels with icons + labels.
                           Auto-built; no prefab needed.
- SimplePlayerRig.cs       Self-contained CharacterController + WASD + mouse look.
                           Builds Camera child if absent.

PLACEMENT / SPAWNING
- PropPlacer.cs            Player click-to-place + assistant programmatic placement.
                           Cube ghost preview, anti-overlap nudge.
- PropPool.cs              Pre-warm N GLBs into hidden cache (heat texture/mesh load).
- RuntimeThumbnailCapture.cs  Renders GLBs to RenderTexture for hotbar icons.
                              Singleton, cached by prop ID.

SESSION / FLOW
- SandboxManager.cs        Top-level orchestrator. Auto-wires missing dependencies.
                           Owns timer (90s + 30s grace), calls AssistantSystem.
- SandboxBootstrap.cs      One-component entry point — drop on empty GameObject.
- HallwayTrigger.cs        Threshold collider — calls SandboxManager.BeginSandbox().
- EndCard.cs               Session-complete reflection card (auto-built canvas).
- AssistantDebugUI.cs      F1 IMGUI overlay — phase, influence, profile, slots.

ASSISTANT
- AssistantSystem.cs       90-second corruption arc.
                           Helping (0–30s) / Suggesting (30–60s) / Overriding (60–90s).
                           Reactive + autonomous placement. Drives PSX glitch.

POST-PROCESSING
- PSXRendererFeature.cs    URP renderer feature — Bayer dither, color quantization.
- PSXPass.cs               ScriptableRenderPass implementation.
                           Glitch intensity driven by AssistantSystem.Influence.
- (Shaders/PSXPost.shader) HLSL shader for the pass.

==============================================================
SCENE SETUP — V2 quickstart
==============================================================

Minimum (zero-wire):
  1. New empty scene.
  2. Create empty GameObject "Bootstrap" → add SandboxBootstrap component.
  3. Press play.

That's it. SandboxBootstrap creates a SandboxManager which auto-creates:
  - SandboxFloor (20x20 plane)
  - PropPlacer (with SculptureSpawner)
  - HotbarController + HotbarUI
  - AssistantSystem
  - SimplePlayerRig (if no MainCamera)
  - RuntimeThumbnailCapture
  - EndCard
  - AssistantDebugUI
  - Directional Sun light

For PSX visuals: open your URP Forward Renderer asset and add the
PSXRendererFeature in the Renderer Features list.

For the hallway → sandbox transition:
  1. Disable _startSandboxImmediately on SandboxManager.
  2. Build hallway geometry leading to a doorway.
  3. Place a trigger collider at the doorway with HallwayTrigger component.
  4. Reference SandboxManager from HallwayTrigger.

Controls:
  WASD            Move
  Shift           Run
  Mouse           Look
  1-5             Select hotbar slot
  Left click      Place active prop on floor
  Tab             Toggle cursor lock (debug)
  F1              Toggle assistant debug overlay
